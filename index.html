História do computador

Primeiros computadores:

  1. Primeira geração: Mark 1 (1944) / Colossus(1964) / ENIAC(1964)
  2. Segunda geração: mudou de valvula para transistores (1959 a 1965)
  3. Terceira geração: Circuuito integrado: Transistor - Capacitores - Resistores (1965 a 1970)
  4. Quarta geração: a partir de 1971 até hoje, começou a surgir os computadores pessoais
  5. Quinta geração: Computadores quanticos, metaverso, inteligência artificial e bitcoins

História da web

Como surgiu a internet:

-Primeiro sistema de comunicação foi o telegrafo com o código morse em 1958.
-Na década de 1950 veio a Guerra fria e com ela a corrida espacial.
-o ENIAC ficava nos Estados Unidos e era usado para fazer calculos balisticos.
-Surgiu a Arpa (Agência de projeto de pequisa avançada) e depois virou DARPA (Defense advanced research projects agency) Com a necessidae de se comunicar surgiu a Arpanet, sua primeira conexão estabelecida foi feita em 29/10/1969 entre a UCLA(University of california) e SRI(Stanford research institute), eles mandaram a palavra login mas só chegou lo.
-A Arpanet foi evoluindo ganhando alguns pontos de coneão até que Vinton Cerf e Robert Kahn criaram o TCP/IP que foi crescendo até serem formadas micro redes, redes mais expecializads
-A Arpa net era para pesquisas cientificas e militares, baaseado nela surgiu a Mil Net era exclusivamente militar, a NFSnet era outra sub net voltada para noticías.
-Em 1989, Tim Berners-lee viu a oportunidade de unir hipertexto com TCP/IP e criar a WWW o primeiro navegador se chamava World Wide Web, em seguida veio o mosaic, netscaipe e assim por diante, em 06 de agosto de 1991 ele públicou a primeira página web que você consegue acessar Clicando aqui

Entendendo comunicação Client X Servidor

Cliente

É quando você consome a informação de algum lugar, é o dispositivo que está consumindo a informação na internet ex: Notbock se conecta na internet com o servidor. O servidor, serve informações para os clientes. Quando você tem um site e publica, você está hospedando o site no servidor, significa que você vai colocar o seu arquivo de internet no servidor.

servidor

Existem vários tipos de servidores:
-Arquivos (Centraliza imagens, àudio, vídeos.. para outros terem acesso)
-Segurança (Firewall)
-Streaming (Faz streaming de dados, muitas vezes de vídeo)
-E-mail (Armazena e gerencia E-mails)
-Web (Servidor para internet, html, css, js ..)
Um servidor pode ser um computador pequeno ou uma sala com vário computadores que são conhecidos como data center que serve vário clientes (Tipo os servidores do Google, Facebook...)

Estrutura básica de um html

O documento HTML sempre inicia com o que chamamos de estrutura básica

  1. Definição do documento (doctype)
  2. Cabeça (head)
  3. Corpo (body)

Doctype - Definindo o documento

Uma coisa importante: SEMPRE deve existir o doctype, que é este código <!DOCTYPE html>.

O doctype não é uma tag HTML, mas uma instrução para o navegador e outros programas que podem ler seu site, que o código encontrado ali é um código HTML. Assim eles sabem o que fazer para mostrar seu site da melhor forma possível. Lembre-se: o doctype é OBRIGATÓRIO e deve ser sempre a PRIMEIRA LINHA do seu documento.

HEAD

Contém informações que não são transpostas visivelmente para o usuário/leitor do documento. São dados implícitos, de uso e controle do documento: vinculação com outros arquivos, aplicação de lógica de programação de scripts e metadados. Na prática, todo o conteúdo do cabeçalho fica delimitado entre a abertura e fechamento tag head.

BODY

Trata-se do documento em si, ou seja, a informação legível para o usuário/leitor do documento. É todo e qualquer texto que se deseja apresentar, assim como toda e qualquer forma de mídia de saída (imagens, sons, miniaplicativos embutidos, conteúdo multimídia, etc). Além disso, toda a apresentação de entrada de dados (formulários) também se aplica neste seção do documento. Na prática, o corpo do documento é delimitado pelo par de tags <body> e </body>.

Este é o preceito básico que deve estar muito bem claro para você: onde as marcações se aplicam, e quais são os resultados deste modelo. Por exemplo: se vocês deseja informar conteúdo textual para saída legível ao usuário do seu sistema web, esta marcação deverá obrigatoriamente estar no bloco do corpo da página. Ainda: para definir qual o tipo de codificação da página (uma meta informação do documento), esta deve obrigatoriamente estar marcada no cabeçalho do mesmo documento.

Dentro do elemento BODY sua estrutura de página terá os elementos semânticos da construção da sua página, onde serão declarados e identificados cabeçalhos, rodapé, conteúdo principal, etc.
